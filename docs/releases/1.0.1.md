# 1.0.1

Security and UX hardening update after the first public release.

## Highlights

- Added a preview-ready Organizations section and navigation flow
- Improved loading behavior while the free-tier backend wakes from sleep
- Refined profile and verification UI for a cleaner, more minimal flow
- Added stronger verification abuse controls and safer error handling
- Added browser and request-level hardening for production traffic

## Changes

- Added `/organizations` page with search, sorting, pagination, and minimal organization blocks
- Added organization seed data for local preview
- Hardened verification request limits and generalized verification error messages
- Removed production `localhost` credentialed CORS allowance
- Added request origin checks for state-changing authenticated API calls
- Added security headers in the web app configuration
- Improved mobile spacing and loading-state behavior

## Notes

- This release is intended as a safer and more polished baseline before broader rollout.
- Organization data can be curated manually before any production exposure.
